# PixelComparison 使用文档

游戏截图视觉回归对比平台:采集模块上报各版本的场景截图,平台对任意两个批次做像素级对比,
输出差异率、SSIM、PSNR 等指标和差异热力图,帮助 QA 快速定位渲染回归。

---

## 1. 系统架构

```
游戏端 / CI 采集模块
      │  POST /api/batches + /screenshots(上报批次与截图)
      ▼
FastAPI 后端 ──── SQLite(批次/截图/基线/对比结果)
      │   对比引擎:Pillow + numpy(差异率 / SSIM / PSNR / 热力图 / 直方图)
      │   静态服务:/images(截图与热力图文件)
      ▼
Vue 3 + Arco Design 前端(批次管理 / 对比查看)
```

| 目录 | 说明 |
|---|---|
| `backend/app/compare.py` | 对比引擎:像素 diff、SSIM、PSNR、热力图、直方图 |
| `backend/app/service.py` | 对比服务(按场景名配对)、基线晋升、状态阈值 |
| `backend/app/models.py` | Batch / Screenshot / Baseline / Comparison / ComparisonItem |
| `backend/app/main.py` | REST API + 静态图片服务 |
| `backend/app/seed.py` | 演示数据生成(可重复执行,会清库重建) |
| `frontend/src/store.js` | 前端全部状态与数据加载(Pinia) |
| `frontend/src/components/` | 页面组件 |

## 2. 安装与启动

### 环境要求

- Python ≥ 3.11
- Node.js ≥ 18

### 后端

```powershell
cd backend
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
.venv\Scripts\python -m app.seed        # 可选:生成演示数据(清库重建)
.venv\Scripts\python -m uvicorn app.main:app --port 8000 --reload
```

- 交互式 API 文档:http://127.0.0.1:8000/docs
- 数据与图片存放在 `backend/data/`(SQLite 库 + 图片目录),删除该目录即彻底重置

### 前端

```powershell
cd frontend
npm install
npm run dev          # 开发:http://localhost:5173(已配置 /api 与 /images 代理)
npm run build        # 生产构建,产物在 dist/
```

## 3. 核心概念

| 概念 | 含义 |
|---|---|
| **批次 Batch** | 一次截图采集运行(项目 + 分支 + 平台),由采集模块上报,平台侧不手动创建 |
| **截图 Screenshot** | 批次内某个场景的一张图,以场景名为配对键 |
| **基线 Baseline** | 被"晋升"为参照标准的批次,带版本号(如 v1.1.5),按项目 + 平台隔离;同版本旧基线自动退役 |
| **对比 Comparison** | 当前批次 × 参照批次的一次对比;参照批次任选,同一批次可与多个参照各比一次 |
| **对比项 ComparisonItem** | 单场景的对比结果 |

### 场景状态判定

按场景名配对两个批次的截图后,每个场景进入以下状态之一:

| 状态 | 条件 | 含义 |
|---|---|---|
| 通过 pass | 差异率 < 0.3% | 视为无变化 |
| 警告 warn | 0.3% ≤ 差异率 < 2% | 轻微差异,建议人工过目 |
| 失败 fail | 差异率 ≥ 2% | 显著差异,疑似渲染回归 |
| 新增 added | 仅当前批次有此场景 | 新场景,确认后可入基线 |
| 缺失 missing | 仅参照批次有此场景 | 当前批次未产出截图,按失败级处理 |

整个对比的状态:任一场景 fail/missing → 失败;任一 warn/added → 警告;否则通过。

> 阈值在 `backend/app/service.py` 顶部的 `FAIL_THRESHOLD` / `WARN_THRESHOLD` 修改;
> "差异像素"的单像素判定阈值在 `backend/app/compare.py` 的 `PIXEL_DIFF_THRESHOLD`(默认任一通道差值 > 8)。

## 4. 界面操作指南

### 4.1 批次对比列表(上方表格)

- 每行是一次**对比**(不是批次本身):批次 ID、分支、参照基线/批次、平台、场景数、配对成功的对比数、状态、平均差异率
- "参照基线 / 批次"列:参照批次若是基线显示版本号(v1.1.5),否则显示批次号(#20240523_2147),悬停可见详情
- 点击行或「查看结果」加载该对比的场景结果
- **发起对比**:右上角按钮,弹窗中选择两个批次:
  - 当前批次:待检查的新批次
  - 参照批次:只列出同项目、同平台的批次,已晋升基线的优先展示并带标签
- **导出列表**:当前列表导出 CSV
- 左侧筛选栏:按项目 / 平台 / 分支 / 基线版本 / 状态过滤,「应用筛选」生效,「清空」重置

### 4.2 场景结果列表(左下)

- 状态 chip(全部 / 失败 / 警告 / 通过,出现时还有 新增 / 缺失)点击过滤,数字为各状态计数
- 搜索框按场景名模糊过滤(300ms 防抖)
- 每行:缩略图、场景名、状态、差异率;点击在右侧查看详情
- 底部分页,默认每页 10 条

### 4.3 对比详情(中部)

- 标题行:场景名、状态、「上一个 / 下一个」(也可用键盘 ← →,顺序为场景名排序)
- 三种视图模式:
  - **三视图**:当前版本 / 参照版本 / 差异热力图并排(热力图越红差异越大)
  - **滑动对比**:两图叠放,拖动分割线或点击任意位置对比(单边缺图时不可用)
  - **原图对比**:只看两张原图
- 缩放:50% – 200%,超出容器可滚动
- 新增 / 缺失场景会显示提示条,缺图侧显示占位

### 4.4 指标面板(右侧)

| 指标 | 含义 |
|---|---|
| 差异率(像素) | 差异像素占总像素的百分比(核心判定指标) |
| 差异像素数 / 总像素数 | 绝对数量 |
| 最大差异(单像素) | 所有像素中最大的通道差值(0–255) |
| 平均差异(仅差异区域) | 只统计差异像素的平均差值 |
| RMS | 均方根误差,整体差异强度 |
| SSIM | 结构相似度,1 为完全一致(演示为全局简化版) |
| PSNR | 峰值信噪比(dB),越高越接近;99 表示完全一致 |
| 通道差异 | R/G/B 各通道的差异像素占比 |
| 直方图对比 | RGB 分布,实线填充 = 当前版本,虚线 = 参照版本 |

## 5. 采集模块接入(批次上报)

批次由外部模块通过 API 上报,流程:建批次 → 逐张传图 → (可选)发起对比。

```bash
# 1. 创建批次(id 可不传,默认用时间戳)
curl -X POST http://127.0.0.1:8000/api/batches \
  -H "Content-Type: application/json" \
  -d '{"id":"20240601_0900","project":"Project_Starfall","branch":"release/1.2.1","platform":"Windows","creator":"CI机器人"}'

# 2. 上传截图(每个场景一张,场景名即配对键,重复上传返回 409)
curl -X POST http://127.0.0.1:8000/api/batches/20240601_0900/screenshots \
  -F "scene_name=01_废弃都市_广场_昼" \
  -F "file=@shot_01.png"

# 3. 发起对比(也可由用户在界面上操作)
curl -X POST http://127.0.0.1:8000/api/comparisons \
  -H "Content-Type: application/json" \
  -d '{"batch_id":"20240601_0900","ref_batch_id":"20240510_0900"}'
```

Python 示例:

```python
import requests

BASE = "http://127.0.0.1:8000"
batch = requests.post(f"{BASE}/api/batches", json={
    "project": "Project_Starfall", "branch": "release/1.2.1",
    "platform": "Windows", "creator": "CI机器人",
}).json()

for scene, path in scenes.items():   # {场景名: 图片路径}
    with open(path, "rb") as f:
        requests.post(f"{BASE}/api/batches/{batch['id']}/screenshots",
                      data={"scene_name": scene}, files={"file": f})

requests.post(f"{BASE}/api/comparisons",
              json={"batch_id": batch["id"], "ref_batch_id": "20240510_0900"})
```

注意:

- 两个批次**平台必须相同**,否则返回 400
- 场景名必须前后版本保持一致,它是唯一的配对键
- `POST /api/comparisons` 当前为同步执行,场景多时请求耗时较长(生产化应改为异步队列,见 §7)

## 6. API 一览

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/api/batches` | 批次列表(可按 project/platform 过滤) |
| POST | `/api/batches` | 上报批次 |
| POST | `/api/batches/{id}/screenshots` | 上传截图(multipart: scene_name + file) |
| GET | `/api/comparisons` | 对比列表(project/platform/branch/baseline/status/q) |
| POST | `/api/comparisons` | 发起对比 `{batch_id, ref_batch_id}` |
| GET | `/api/comparisons/{id}/scenes` | 场景结果(status/q/page/page_size,含各状态计数) |
| GET | `/api/items/{id}` | 单场景详情(指标 + 图片 URL + 前后场景 id) |
| GET | `/api/baselines` | 基线列表 |
| GET | `/api/meta` | 筛选器选项 |

## 7. 常见问题

**端口被占用** — 后端默认 8000、前端 5173,改端口时同步更新 `frontend/vite.config.js` 的代理目标和 `backend/app/main.py` 的 CORS 白名单。

**想重置演示数据** — `cd backend && .venv\Scripts\python -m app.seed`(会清库并重新生成全部图片),或直接删除 `backend/data/` 目录。

**上传同名场景报 409** — 同一批次内场景名唯一;重传请先删批次或换批次号。

**对比请求很慢** — 同步执行所致,场景数 × 单张对比约几十毫秒;生产化改造方向:
PostgreSQL 替换 SQLite、MinIO/S3 存图、Celery + Redis 异步跑对比(立即返回 pending,
worker 完成后更新状态)、`compare.py` 换 OpenCV / scikit-image 的 windowed SSIM。

**界面上的"基线管理 / 项目设置"页签** — 当前为占位,规划功能:基线列表与晋升 / 退役操作
(后端 `promote_baseline` 已实现,差 UI 和接口)、逐场景接受差异更新基线、默认参照策略配置。
